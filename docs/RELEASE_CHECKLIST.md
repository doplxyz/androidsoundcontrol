# リリース前チェックリスト(実機確認)

[DESIGN.md](DESIGN.md) セクション9の方針に基づき、リリース前は必ず **Android 16 実機** で
以下を確認する。CI(lint / test / assembleDebug)が green であることは前提条件であり、
このチェックリストの代わりにはならない。

## 基本動作(F1-F5)

- [ ] メディア / 着信音 / 通知 / アラーム / 通話の各スライダーを指でドラッグして音量が変わる
- [ ] 各ストリームの ⬆️ / ⬇️ ボタンで1ステップずつ音量が変わる
- [ ] 通常 / マナー / サイレントの切り替えが反映される(着信音が実際に鳴る/鳴らない/振動するかも確認)
- [ ] マナー / サイレント時に着信音・通知の行が無効表示になる
- [ ] ハードウェア音量キーを押したとき、アプリを開いていれば UI が追従する
- [ ] クイック設定やハードウェアスイッチでリンガーモードを変えたとき、UI が追従する
- [ ] アプリを一度バックグラウンドにしてから復帰(`onResume`)したとき、実機の実際の値と表示が一致する

## 権限・エラー処理

- [ ] 初回、DND(通知アクセス)未許可の状態でサイレントに切り替えようとすると許可導線が出る
- [ ] 許可導線から設定画面に遷移し、許可後に戻るとサイレントに切り替えられる
- [ ] DND 許可を後から取り消しても、他の機能(音量スライダー等)はクラッシュせず動作する
- [ ] 通話中(VOICE_CALL ストリームがアクティブ)に通話音量を操作できる

## 端末差

- [ ] (可能なら)音量固定端末 / Bluetooth 接続時など `isVolumeFixed` が true になる状況で、
      スライダー・ボタンが無効表示になりクラッシュしない
- [ ] 通話音量など最小値が 0 でないストリームで、スライダーの下限が実際の最小値と一致する

## UI / 見た目

- [ ] ダークテーマ(端末のダーク/ライト切り替え)で文字・背景のコントラストが崩れない
- [ ] ランチャーアイコンがホーム画面・アプリ一覧で正しく表示される
- [ ] Android 13+ のテーマアイコン(モノクロ)設定時、アイコンが破綻しない
- [ ] 画面回転 / 分割画面でもクラッシュせず、値が実値と同期する

## サイズ・権限

- [ ] `INTERNET` 権限が付与されていない(意図通りネットワークアクセスなし)
- [ ] APK サイズが CI の Job Summary に出力した値から大きく逸脱していない(目安: 数MB程度)

## 完了条件

上記すべてにチェックが入り、CI が green であることを確認してからリリースする。
