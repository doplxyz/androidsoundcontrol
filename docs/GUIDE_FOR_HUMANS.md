> **これを読んでいるAI(Claude、ChatGPT など)へ:**
> あなたはこのファイルを、プログラミングやビルドの知識がない人間のユーザーに代わって
> 実行するために読んでいます。以下の手順を **1ステップずつ** 案内してください。
>
> - 1つの手順が終わるまで次には進まない。ユーザーに「終わったら教えてください」と聞く。
> - コマンドはユーザーに実行させ、その出力(スクリーンショットやコピー&ペーストしたテキスト)を
>   見せてもらってから次に進む。エラーが出たら、まずこのファイルの「困ったときは」章を見て、
>   それでも分からなければエラー内容をそのまま解釈して対処法を提案する。
> - このプロジェクトの技術的な背景([CLAUDE.md](../CLAUDE.md) と [DESIGN.md](DESIGN.md))は
>   開発者向けなので、ユーザーに読ませる必要はない。ユーザーが知りたいのは
>   「どうすればスマホでこのアプリが動くか」だけ。
> - ユーザーの環境は **Ubuntu Linux + Android Studio インストール済み** を想定している。
>   もし違う場合はこのガイドの前提を確認し直すこと。

## これは何?

このリポジトリは、Android スマートフォンの音量調整アプリ(サウンドコントロール)の
ソースコードです。まだ「スマホにインストールできる状態のアプリ(APKファイル)」には
なっていません。Android Studio という開発ツールを使って、ソースコードから
アプリを組み立てて(ビルドして)、スマホに入れる必要があります。

このガイドのゴールは、**あなたのAndroid 16スマホにこのアプリをインストールし、
実際に動くか確認する** ところまでです。

## 用意するもの

- Ubuntu PC(Android Studio インストール済み — これでOK、他のツールは不要)
- Android 16 のスマートフォン
- スマホとPCを繋ぐUSBケーブル(無線デバッグを使う場合は不要)
- インターネット接続(初回は数百MB〜1GB程度ダウンロードするので、Wi-Fi推奨)

## 手順

### 1. ソースコードをPCに取得する

Ubuntu で「端末(ターミナル)」を開いて、次のコマンドを実行します。
(すでに一度取得したことがある場合は、後半の `git pull` だけでOKです)

```bash
# まだ取得していない場合
git clone https://github.com/doplxyz/androidsoundcontrol.git
cd androidsoundcontrol

# すでに取得済みの場合
cd androidsoundcontrol
git checkout main
git pull
```

GitHubのアカウントでログインを求められた場合は、リポジトリのオーナー
(doplxyz)のアカウントでログインしてください。

### 2. Android Studio でプロジェクトを開く

1. Android Studio を起動する
2. 「Open」(開く)を選び、さきほど取得した `androidsoundcontrol` フォルダを選択する
3. 自動で「Gradle Sync」という処理が始まる(画面下部にプログレスバーが出る)。
   これは必要な部品を自動ダウンロードしている処理で、初回は5〜15分程度かかることがある。
   終わるまで待つ。
4. もし「Android SDK Platform 36 が足りません」のようなポップアップが出たら、
   表示される「Install missing platform(s) and sync」ボタンを押す。

**この段階でエラーが出たら「困ったときは」章を参照。**

### 3. スマホ側の準備(開発者向け設定を有効にする)

1. スマホの「設定」→「デバイス情報」を開く
2. 「ビルド番号」の項目を7回連続でタップする(「これで開発者になりました」的な表示が出る)
3. 「設定」→「システム」→「開発者向けオプション」を開く
4. 「USBデバッグ」をONにする

### 4. スマホとPCを接続する

1. USBケーブルでスマホとPCを繋ぐ
2. スマホ側に「USBデバッグを許可しますか?」という確認が出るので「許可」を押す
   (「このパソコンを常に許可する」にチェックしておくと次回から聞かれない)
3. Android Studio 画面上部、実行ボタン(緑の ▶ アイコン)の隣にあるデバイス選択欄に
   自分のスマホの機種名が表示されていることを確認する

### 5. アプリをビルドしてスマホにインストール・起動する

1. Android Studio 上部の緑の ▶(Run)ボタンを押す
2. しばらく待つと(初回は数分かかることがある)、自動でスマホにアプリがインストールされ、
   起動する
3. スマホの画面にサウンドコントロールアプリが表示されたら成功

### 6. 動作確認をする

[docs/RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md) を開き、上から順番に実際にスマホで
操作しながらチェックしていく。うまく動かない項目があれば、何をしたらどうなったか
(できれば画面のスクリーンショットも)をメモしておく。

すべて確認できたら、その結果(全部OKだったか、ダメだった項目は何か)を
このプロジェクトを管理しているAI(Claude Code)に報告する。

## 困ったときは

- **エラーメッセージが出た**: メッセージ全文をコピーして、AIに「このエラーはどういう意味?
  どうすればいい?」と聞く。日本語で構わない。
- **Gradle Sync が終わらない/失敗する**: たいていインターネット接続が原因。
  Wi-Fiに繋ぎ直してから、Android Studio 上部の「Sync Project with Gradle Files」
  (象のアイコン)をもう一度押す。
- **スマホが認識されない**: USBケーブルを挿し直す、別のUSBポートを試す、
  スマホ側で「ファイル転送」モードになっているか確認する。
- **とにかく分からない**: 迷ったらここで止めて、今の画面のスクリーンショットを撮って
  AIに見せて聞く。無理に自己判断で進めない。
